ðŸ§± OOP in C++

4. Class Structure

class MyClass {
private:
    int x;

public:
    MyClass(int val) : x(val) {}
    void display() const { cout << x << endl; }
    ~MyClass() {}
};

5. Inheritance

class Base {
public:
    virtual void show() {}
};

class Derived : public Base {
public:
    void show() override {}
};
  • Access: public, protected, private
  • Use virtual destructors in base class

13. Move Semantics

class A {
public:
    A(A&&) noexcept;
    A& operator=(A&&) noexcept;
};

14. Rule of 0 / 3 / 5

  • 0: Compiler handles everything
  • 3: Define copy ctor, assignment, destructor
  • 5: Add move ctor and move assignment

ðŸ’¨ Common Pitfalls

MistakeExample / Fix
Dangling PointersUse smart pointers
Memory LeaksUse RAII or delete
Object SlicingUse virtual functions
Not Using constPrefer const everywhere
Undefined BehaviorInitialize variables, avoid invalid access
Copy vs ReferenceUse const references for large objects
Multithreading IssuesAvoid shared mutable state
Overusing MacrosUse constexpr, inline
Missing Virtual DestructorsUse virtual ~Base()

Other Pitfalls

  • Returning reference to local
  • Improper exception handling
  • Confusion between reference, pointer, copy
  • Off-by-one errors
  • Mismatched new/delete
  • Avoid #define for constants
